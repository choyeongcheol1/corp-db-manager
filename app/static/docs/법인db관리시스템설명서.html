법인 DB 관리 시스템

다중 DB 서버 환경에서 법인(테넌트) DB를 등록·생성·복제·문서화·모니터링하기 위한 웹 기반 통합 운영 솔루션

중앙관리 프로비저닝 스키마 정의서 데이터 복사 모니터링

3. 지원 범위 (L1~L3)

DBMS별로 "지원"의 의미가 다를 수 있으므로, 기능 지원 범위를 레벨(L1~L3)로 구분합니다.

레벨 정의

레벨정의대표 기능
L1 연결/조회서버 연결 및 메타 조회 중심서버 등록, 연결 테스트, DB 목록, 테이블/컬럼 조회
L2 작업문서화/복사 등 운영 작업 지원정의서(Excel) 추출, 데이터 복사(제약 포함)

부록 A. DBMS별 SQL 예시

주요 작업에 사용되는 DBMS별 SQL 구문입니다.

A.1 DB 목록 조회

MSSQL
SELECT name, state_desc, create_date FROM sys.databases WHERE database_id > 4;
PostgreSQL
SELECT datname, pg_size_pretty(pg_database_size(datname)) FROM pg_database;
MySQL
SHOW DATABASES;

A.2 테이블 컬럼 조회

MSSQL
SELECT c.name, t.name AS type, c.max_length, c.is_nullable
FROM sys.columns c JOIN sys.types t ON c.user_type_id = t.user_type_id
WHERE c.object_id = OBJECT_ID('dbo.TableName');
PostgreSQL
SELECT column_name, data_type, is_nullable
FROM information_schema.columns
WHERE table_name = 'table_name';

부록 C. FAQ

자주 묻는 질문과 답변입니다.

Q
DB 생성 후 접속이 안 됩니다.
A. 생성된 DB 계정의 비밀번호를 확인하세요. 기본적으로 프로비저닝 설정에서 지정한 비밀번호가 적용됩니다. 또한, 방화벽에서 클라이언트 IP가 허용되어 있는지 확인하세요.
Q
데이터 복사 중 취소하면 어떻게 되나요?
A. 배치 트랜잭션 정책인 경우 현재 진행 중인 배치까지만 커밋되고 중단됩니다. 전체 트랜잭션 정책인 경우 모든 작업이 롤백됩니다.
Q
Oracle은 왜 프로비저닝이 지원되지 않나요?
A. Oracle은 라이선스 정책과 아키텍처 특성상 스키마 기반 멀티테넌트 구조를 권장합니다. 향후 버전에서 스키마 복제 방식으로 지원 예정입니다.
Q
테이블 정의서 내보내기 시 일부 테이블이 누락됩니다.
A. 시스템 테이블이나 권한이 없는 테이블은 제외됩니다. 또한, 200개 초과 시 자동으로 분할되므로 여러 파일을 확인하세요.
