신규 법인 생성 시나리오

실무 기준 가이드

개요

신규 법인(고객사) 발생 시 해당 법인 전용 데이터베이스를 생성하는 프로세스입니다.

전제 조건

항목 설명
참조 DB 기준이 되는 템플릿 DB가 존재해야 함 (예: ACC_STANDARD)
법인코드 고유한 법인코드가 확정되어야 함 (예: NEWCORP)
서버 선택 신규 DB를 생성할 타겟 서버가 결정되어야 함
권한 admin 또는 operator 역할 필요

시나리오 상세

1 법인 정보 입력

화면: /corps/create 또는 /corps/create/{server_id}

1. 소스 서버/DB 선택

  • 소스 서버: 참조할 템플릿 DB가 있는 서버 선택
  • 소스 DB: 복제 기준이 될 DB 선택 (예: ACC_STANDARD)
  • 소스 DB의 스키마(테이블 구조, 프로시저 등)를 복제함

2. 타겟 서버 선택

  • 신규 DB를 생성할 서버 선택
  • 소스 서버와 동일하거나 다른 서버 선택 가능

3. 법인 정보 입력

  • 법인코드: 고유 식별자 (예: NEWCORP)
    → DB명으로 사용됨: ACC_{법인코드} → ACC_NEWCORP
  • 법인명: 회사명 (예: 신규법인 주식회사)
  • 사업자번호: 10자리 (예: 123-45-67890) - 선택사항

4. 다음 단계 버튼 클릭

2 SQL 미리보기 및 실행

화면: /corps/create/preview

1. 생성될 SQL 확인

시스템이 자동 생성한 SQL 스크립트 표시:

-- 1. 데이터베이스 생성
CREATE DATABASE [ACC_NEWCORP]
ON PRIMARY (
    NAME = N'ACC_NEWCORP',
    FILENAME = N'D:\MSSQL\Data\ACC_NEWCORP.mdf',
    SIZE = 100MB,
    FILEGROWTH = 64MB
)
LOG ON (
    NAME = N'ACC_NEWCORP_log',
    FILENAME = N'D:\MSSQL\Log\ACC_NEWCORP_log.ldf',
    SIZE = 50MB,
    FILEGROWTH = 64MB
)

-- 2. 로그인 계정 생성
CREATE LOGIN [ACC_NEWCORP] WITH PASSWORD = 'DB@1234'

-- 3. DB 사용자 생성 및 권한 부여
USE [ACC_NEWCORP]
CREATE USER [ACC_NEWCORP] FOR LOGIN [ACC_NEWCORP]
ALTER ROLE [db_owner] ADD MEMBER [ACC_NEWCORP]

-- 4. 테이블 생성 (소스 DB 스키마 복제)
-- (테이블 구조, 인덱스, 제약조건 등)

-- 5. 프로시저/함수 생성
-- (소스 DB의 프로시저, 함수 복제)

-- 6. 기초 데이터 입력
-- (공통코드, 메뉴, 권한 등 필수 데이터)

-- 7. 관리자 계정 생성
INSERT INTO TB_USER (USER_ID, USER_NAME, PASSWORD, ...)
VALUES ('admin', '관리자', 'Admin@1234', ...)

2. SQL 수정 (선택사항)

  • 필요 시 SQL 직접 수정 가능
  • 파일 경로, 초기 용량, 비밀번호 등 변경

3. SQL 다운로드 (선택사항)

  • .sql 파일로 다운로드
  • 백업 또는 수동 실행용

4. 실행 버튼 클릭

  • SQL 순차 실행
  • 진행 상황 실시간 표시

3 생성 완료

화면: 생성 완료 결과 표시

1. 생성 결과 확인

  • DB명: ACC_NEWCORP
  • 서버: 운영서버 (192.168.1.100:1433)
  • DB 계정: ACC_NEWCORP / DB@1234
  • 관리자 계정: admin / Admin@1234
  • 생성 소요 시간

2. 후속 작업 선택

  • 데이터 복사하기: 참조 DB에서 추가 데이터 복사 필요 시
  • 법인 목록으로: 생성된 법인 확인
  • 신규 생성하기: 다른 법인 추가 생성

데이터 복사 (선택사항)

법인 생성 후 추가 데이터 복사가 필요한 경우:

화면: /copy-data/{server_id}

1. 소스/타겟 선택

  • 소스: 참조 DB (예: ACC_STANDARD)
  • 타겟: 신규 생성된 DB (예: ACC_NEWCORP)

2. 테이블 선택

  • 복사할 테이블 선택 (기초코드, 설정 등)
  • 예: TB_COM_CODE, TB_MENU, TB_ROLE, TB_AUTH

3. 복사 실행

  • 선택한 테이블 데이터 복사
  • 결과 확인 (성공/실패, 복사 건수)

생성 후 확인 사항

확인 항목 방법
DB 생성 확인 법인 목록에서 확인
테이블 확인 테이블 현황에서 확인
접속 테스트 SSMS 등으로 DB 계정 접속 테스트
애플리케이션 연동 실제 서비스 연결 테스트

주의사항

1. 법인코드 중복 불가

  • 이미 존재하는 법인코드는 사용 불가
  • DB명이 중복되면 생성 실패

2. 디스크 용량 확인

  • 생성 전 타겟 서버 디스크 여유 공간 확인
  • 최소 500MB 이상 권장

3. SQL 수정 시 주의

  • 구문 오류 시 생성 실패
  • 수정 전 원본 SQL 다운로드 권장

4. 운영 시간 고려

  • 대용량 스키마 복제 시 시간 소요
  • 가급적 업무 시간 외 작업 권장

활동 로그 기록

생성 과정에서 자동으로 기록되는 로그:

시점 액션 내용
생성 시작 CREATE_DB DB 생성 시작: ACC_NEWCORP
생성 완료 CREATE_DB DB 생성 완료: ACC_NEWCORP (법인: NEWCORP)
데이터 복사 COPY_DATA 데이터 복사: ACC_STANDARD → ACC_NEWCORP

문제 해결

문제 원인 해결 방법
DB 생성 실패 권한 부족 sa 또는 sysadmin 권한 확인
파일 경로 오류 경로 없음 서버의 데이터/로그 경로 확인
스키마 복제 실패 소스 DB 접근 불가 소스 DB 연결 상태 확인
로그인 생성 실패 중복 로그인 기존 로그인 삭제 후 재시도
